# Evaluation Rubric

## 1) 가중치 (확정)
- 설계 의도 설명 능력: **60점**
- 코드 품질: **10점**
- AI 활용 능력: **30점**

총점 100점 체계.

## 2) 세부 항목

### A. 설계 의도 설명 능력 (60)
- 문제 맥락 정의와 요구사항 해석
- 아키텍처 선택 이유와 대안 비교
- 인프라 의사결정 근거(비용/확장성/운영성)
- 장애/리스크 대응 전략
- 트레이드오프 설명의 일관성

### B. 코드 품질 (10)
- 구조 가독성
- 테스트/검증 흔적
- 유지보수성(모듈성, naming, separation)

### C. AI 활용 능력 (30)
- AI를 어디서 어떻게 썼는지 명확성
- AI 출력 검증/교차검증 방식
- 프롬프트/도구 사용의 생산성 기여와 한계 인식

## 3) 인터뷰 중 신호 예시
- 설계 의도: “왜 이 구조를 택했는지”를 수치/운영경험 기반으로 답하는가
- 코드 품질: 레포 구조와 규칙이 설명과 일치하는가
- AI 활용: 도구 의존이 아니라 검증 루프를 설명하는가

## 4) 리포트 출력 규칙
- 점수만 주지 않고 각 축별 `근거 문장`을 반드시 함께 출력
- 개선점은 “다음 재체험에서 바꿔야 할 행동”으로 제시
