# Product Scope

## 1) 배경
기존 훈련 흐름은 “리포트 확인 후 종료”에 가깝다. 사용자가 실제로 실력을 올리려면, 점수표가 아니라 **면접 순간을 다시 체험하고 답변을 바꿔보는 루프**가 필요하다.

## 2) 목표
- 훈련센터를 과제형 UI가 아닌 **재체험형 면접 스튜디오**로 재정의
- 리포트 각 순간(질문/답변/톤/태도)을 재생하고 즉시 재면접 가능
- 공개 Git 레포를 기반으로 아키텍처/인프라 설계 의도를 방어하는 실전 면접 제공

## 3) 이번 개편의 확정 사항
1. 훈련센터는 리포트 기반 재체험 공간이다.
2. Git 레포 입력은 공개 레포만 가능하다고 명시한다.
3. 평가 축은 설계 의도 60%, 코드 품질 10%, AI 활용 30%.
4. 디펜스 질문 범위에 CI/CD, 배포, 모니터링, 장애대응을 포함한다.
5. 이번 단계는 문서 전면 개편까지 수행한다.

## 4) 포함 범위
- 훈련센터 정보구조(IA), 사용자 플로우, API/데이터모델 재정의
- 리플레이 시뮬레이션 모드와 포트폴리오 디펜스 모드의 인터뷰 엔진 설계
- 평가 루브릭/거버넌스/운영 지표 재설계

## 5) 제외 범위
- 실제 코드 구현(백엔드/프론트)과 마이그레이션 실행
- private repo 연동 및 권한 위임
- RAG 실 인덱싱/벡터 운영

## 6) 성공 지표
- 리포트 -> 훈련센터 진입률
- 재체험 세션 완료율
- 동일 질문 재응답 개선률
- 포트폴리오 디펜스 세션 생성률
