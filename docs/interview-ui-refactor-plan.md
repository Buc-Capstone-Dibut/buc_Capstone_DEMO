# AI 면접 UI 리팩토링 후속 기획

## 1. 포트폴리오 디펜스 셋업 플로우화

현재 `/interview/training`은 훈련 센터 랜딩과 레포 입력이 한 화면에 섞여 있다. 다음 리팩토링에서는 공고/직무 기반 면접과 같은 셋업 플로우 구조로 분리한다.

- 진입 화면: `/interview/training`은 포트폴리오 디펜스 소개와 최근 기록 허브로 축소
- 셋업 화면: `/interview/training/setup` 또는 `/interview/portfolio/setup`
- 단계 구성:
  1. 레포 입력
  2. 레포 구조 분석 결과 확인
  3. 디펜스 브리프 최종 확인
  4. 10분 화상 디펜스 시작
- 상태 관리: 공고/직무 설정과 별도 store를 두되, 세션 시작 preflight와 10분 고정 정책은 기존 면접 플로우와 통일
- 디자인 원칙: 단일 3D 오브젝트 에셋, 기술 로고 칩, 분석 플로우 시각화, 카드 안 카드 구조 최소화

## 2. 면접 결과 리포트 UI

결과 리포트는 점수 나열보다 “어떤 근거로 개선해야 하는지”를 먼저 보이게 구성한다.

- 상단: 총평, 핵심 점수, 다음 액션
- 본문: 질문별 답변 타임라인, 평가 근거, 개선 예시
- 기술 스택: 이번 작업에서 만든 기술 로고 칩 재사용
- 포트폴리오 디펜스 리포트: 설계 의도, 코드 품질, AI 활용 비중을 시각적으로 분리

## 3. 내 면접 기록 UI

면접 기록은 단순 리스트가 아니라 반복 훈련 이력을 비교하는 화면으로 바꾼다.

- 필터: 공고 기반, 직무 기반, 포트폴리오 디펜스
- 카드/테이블 전환: 최근 기록은 카드, 장기 이력은 테이블
- 비교: 동일 직무/동일 레포 반복 면접의 점수 변화
- 상태: 분석 대기, 분석 완료, 재시도 가능 여부를 명확히 표시
