# 랭크(등급) 시스템 정책서

최종 수정일: 2026-03-02

## 1. 목적

- 커뮤니티와 워크스페이스에서의 실제 기여를 점수화한다.
- 초기 진입 장벽을 낮추기 위해 `Unranked`에서 시작하고, 소량의 활동만으로 `Bronze` 진입이 가능하도록 설계한다.
- 스팸/어뷰징은 점수 회수 및 차감으로 억제한다.

## 2. 등급 구간

| 등급 | 점수 구간 |
|---|---:|
| Unranked | 0 ~ 29 |
| Bronze | 30 ~ 149 |
| Silver | 150 ~ 399 |
| Gold | 400 ~ 899 |
| Platinum | 900 ~ 1799 |
| Diamond | 1800+ |

### 승급/강등 규칙

- 승급: 기준 점수 도달 즉시 자동 반영
- 강등: 기본적으로 없음(단, 운영 제재 차감으로 구간 하락 가능)

## 3. 점수 획득 규칙

| 이벤트 | 점수 | 비고 |
|---|---:|---|
| Q&A 답변 채택 | +25 | 질문 작성자만 채택 가능, 1회만 적용 |
| 답변 좋아요 | +2 | 일일 상한 적용 권장 |
| 질문 작성 | +3 | 저품질/삭제 시 회수 가능 |
| 워크스페이스 태스크 완료 | +1 | 일일 상한 적용 권장 |
| 워크스페이스 문서 작성/주요 수정 | +2 | 단순 오탈자 수정 제외 가능 |
| 프로젝트(워크스페이스) 정상 종료 참여 | +20 | 종료 시점 멤버 기준 |
| 프로필 80% 이상 완성(1회) | +10 | 최초 1회만 지급 |

## 4. 점수 차감/회수 규칙

| 사유 | 점수 | 비고 |
|---|---:|---|
| 운영진 확정 제재(스팸/도배/욕설 등) | -30 ~ -100 | 사안별 수동 결정 |
| 삭제된 저품질 콘텐츠 | 지급분 회수 | 질문/답변/댓글 삭제 시 이벤트 역적용 |
| 자기 채택/어뷰징 패턴 | 미지급 또는 회수 | 중복/비정상 패턴은 로그 기록 |

## 5. 어뷰징 방지 원칙

- 이벤트 단위 중복 지급 방지(`dedupe_key`)를 적용한다.
- 짧은 시간 내 비정상 반복(좋아요 품앗이, 상호 채택 등)은 관리자 검토 대상으로 분류한다.
- 점수 정책 변경 시 기존 점수는 원칙적으로 유지하고, 신규 이벤트부터 변경 규칙을 적용한다.

## 6. UI 노출 원칙

- 프로필: 등급 배지 + 현재 점수 + 다음 등급까지 남은 점수
- 커뮤니티/댓글/사용자 카드: 등급 배지만 간단 노출
- 상세 적립 이력: 프로필 내 `활동 점수` 영역에서만 노출

## 7. 운영 메모

- 현재 채택 취소/변경 정책은 **미허용**(채택은 확정 처리)
- 추후 등급 보상(뱃지, 권한, 노출 우선순위)은 별도 정책서로 분리 관리
- 초기 운영 2주 동안 데이터 확인 후 점수값 미세 조정 권장
